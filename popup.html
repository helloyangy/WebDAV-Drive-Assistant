版本更新（0.1.3 → 0.1.4）
      本次更新修复 AI 备份误报失败问题，并优化大文件上传窗口体验。
      重点更新
      • 修复 MKCOL 423 Locked 被错误当作备份失败的问题，不再弹出多余的失败提示。
      • AI 备份新增文件去重机制，避免同一文件被并发上传两次。
      • 大文件上传窗口样式全面优化：更紧凑的布局、虚线拖拽区选择文件、移除冗余按钮和说明文字。
      修复与优化
      • ensureWebDavDir 新增对 423 状态码的容忍，视为目录已存在。
      • 大文件上传窗口从 560×620 缩小至 360×300，去除弹窗套弹窗感。
      What's new (0.1.3 → 0.1.4)
      This update fixes false AI backup failure reports and redesigns the large file upload window.
      Highlights
      • Fix MKCOL 423 Locked being incorrectly reported as a backup failure, eliminating spurious error toasts.
      • Add file deduplication to AI backup to prevent the same file from being uploaded concurrently.
      • Redesigned large file upload window: compact layout, dashed drop-zone file picker, removed redundant buttons and text.
      Fixes & improvements
      • ensureWebDavDir now tolerates 423 status code, treating it as directory already exists.
      • Large file upload window shrunk from 560×620 to 360×300, removing the nested dialog feel.
